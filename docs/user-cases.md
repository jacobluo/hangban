# User Case

## 1. 使用者与共同前提

首期只有一个系统角色：未登录访客。不同使用动机通过 User Case 表达，不建立账号、权限或个人资料模型。

共同前提：

- 站点可公开访问。
- 地图和机场静态数据已加载。
- 实时数据可能完整、部分降级或暂时不可用。
- 所有时间、来源和覆盖度均按统一产品规则展示。

## 3. 详细 User Case

### UC-01 浏览全球实时地图

**目标**

访客进入站点后，快速了解当前地图范围内有哪些航班及其大致运动状态。

**主流程**

1. 系统打开全球实时地图。
2. 系统根据默认地图视野订阅实时数据。
3. 地图展示当前视野内的航班标记。
4. 航班位置按更新周期平滑变化。
5. 访客拖动或缩放地图，系统更新订阅范围。

**替代流程**

- 当前视野没有航班时，显示「当前视野暂无航班」，并保留搜索和缩放操作。
- 数据源部分不可用时，继续展示可用来源的数据，并显示降级状态。
- 所有实时来源不可用时，保留基础地图和机场信息，并说明实时服务暂时不可用。

**验收条件**

- 地图交互不会因航班标记数量明显阻塞。
- 客户端只接收当前视野或授权范围内的数据。
- 页面可见最后更新时间和实时服务状态。

### UC-02 搜索航班、机场或城市

**目标**

访客通过一个搜索入口定位航班、机场或城市。

**主流程**

1. 访客输入航班号、IATA/ICAO 代码、机场名称或城市名称。
2. 系统在短暂防抖后返回分组结果。
3. 结果显示对象类型和必要的识别信息。
4. 访客选择结果。
5. 系统移动地图并打开相应的航班详情或机场概览。

**替代流程**

- 关键字为空时不发起搜索。
- 没有匹配项时显示空结果说明，并允许修改关键字。
- 搜索接口失败时保留输入内容，提供重试操作。

**验收条件**

- 航班、机场和城市在结果中可区分。
- 键盘和触摸操作均可完成选择。
- 搜索失败不会清空已输入内容。

### UC-03 查看航班摘要

**目标**

访客选择地图上的航班后，了解航班身份、位置和主要运动指标。

**主流程**

1. 访客选择航班标记。
2. 系统突出航班标记和可用航迹。
3. 桌面端打开右侧面板；手机端打开底部抽屉。
4. 系统展示航班号、航空公司、机型、起终点、高度、地速、航向和更新时间。
5. 访客可进入完整详情。

**替代流程**

- 某些字段缺失时显示「未获得数据」或隐藏非关键指标。
- 航班在查看期间离开订阅区域时，详情保留最后位置并说明更新时间。

**验收条件**

- 选中对象在地图和详情中一致。
- 不展示虚构的时刻、延误或到离港状态。
- 实时字段带有更新时间和来源说明。

### UC-04 查看完整航班详情

**目标**

访客查看航班摘要以外的公开字段，并理解数据质量。

**主流程**

1. 访客从航班摘要进入完整详情。
2. 系统展示航班身份、航空器、起终点、实时指标、数据源和覆盖度。
3. 访客返回地图，地图保持此前视野和选中状态。

**验收条件**

- 返回操作不会丢失地图上下文。
- 字段分组便于在桌面端和手机端扫读。
- 推断字段与来源直接提供的字段在语义上可区分。

### UC-05 探索机场

**目标**

访客按名称、代码、城市、附近或热门入口找到机场。

**主流程**

1. 访客进入「机场」。
2. 系统展示机场搜索、区域筛选和当前视野机场列表。
3. 访客选择机场。
4. 地图定位机场并突出机场标记。
5. 系统展示代码、名称、城市、国家、坐标、海拔和覆盖情况。

**替代流程**

- 机场代码存在歧义时，同时展示 IATA、ICAO、城市和国家。
- 当前视野没有机场时，允许搜索或扩大地图范围。

**验收条件**

- 机场搜索和地图选择进入同一详情状态。
- 机场静态资料与实时来源状态分开表达。
- 手机端可在底部抽屉内完成选择。

### UC-06 查看机场周边航班

**目标**

访客观察指定机场一定半径内的实时航班。

**主流程**

1. 访客打开机场概览。
2. 系统以机场坐标为中心查询周边实时航班。
3. 系统展示周边航班数量和列表。
4. 访客选择列表中的航班。
5. 系统进入该航班的地图选中态。

**替代流程**

- 周边没有实时航班时显示空状态，不推断机场停止运行。
- 实时服务不可用时仍展示机场静态资料，并隐藏或禁用周边航班区域。

**验收条件**

- 页面明确说明周边航班不等同于到港或离港班次。
- 半径和更新时间有统一产品定义。
- 航班列表与地图标记保持一致。

### UC-07 探索航线

**目标**

访客选择起点和终点机场，查看当前匹配的在途航班。

**主流程**

1. 访客进入「航线」。
2. 访客选择出发机场和到达机场。
3. 系统展示起终点、大圆航线、距离和数据说明。
4. 系统查询当前匹配的在途航班。
5. 系统在地图和列表中展示匹配航班及其飞行进度。
6. 访客选择航班进入航班摘要。

**替代流程**

- 起终点相同时，系统提示重新选择，不发起查询。
- 当前没有匹配航班时，保留航线和机场资料，显示空状态。
- 匹配依据不足时不展示确定结果，并说明数据覆盖不足。

**验收条件**

- 起终点输入可清除和重新选择。
- 航线结果明确说明不是官方完整班次表。
- 在途数量、列表和地图标记一致。

### UC-08 调整地图筛选与图层

**目标**

访客减少不相关信息，聚焦特定高度、航空公司、航班状态或地图图层。

**主流程**

1. 访客打开筛选或图层面板。
2. 访客修改条件并应用。
3. 系统更新地图订阅和可见图层。
4. 页面显示已启用条件，并支持一键清除。

**验收条件**

- 只提供统一数据模型能够稳定支持的筛选条件。
- 修改条件时避免为每次点击立即建立新连接。
- 筛选为空结果时有明确说明。

### UC-09 判断数据是否可靠和新鲜

**目标**

访客判断当前信息是否足以作为实时观察参考。

**主流程**

1. 页面持续显示总体数据状态和最后更新时间。
2. 访客打开数据状态区域。
3. 系统展示各来源健康状态、最近成功采集时间和降级说明。
4. 过期数据降低视觉强调，并显示最后有效时间。

**验收条件**

- 「实时」「延迟」「过期」「不可用」具有固定定义。
- 单个来源失败不会被描述为所有数据不可用。
- 界面不暴露供应商密钥、内部错误堆栈或调试信息。

### UC-10 在手机端完成核心任务

**目标**

访客在手机浏览器中完成地图浏览、搜索、机场探索、航线探索和航班详情查看。

**主流程**

1. 访客打开手机端页面。
2. 地图和顶部搜索正常显示。
3. 访客通过地图或搜索选择对象。
4. 底部抽屉展示对象信息。
5. 访客展开、收起或切换详情，并继续操作地图。

**验收条件**

- 核心可点击区域不小于 44 × 44 px。
- 不依赖悬停完成关键操作。
- 抽屉内容不被安全区域或浏览器工具栏遮挡。
- 常用手机宽度下没有关键内容横向溢出。
- 屏幕阅读器可以识别搜索、地图控制、列表和抽屉标题。

## 4. 跨 User Case 规则

### 可访问性

- 交互控件提供可访问名称和清晰焦点状态。
- 状态不只依赖颜色表达。
- 地图外提供可键盘访问的搜索结果和对象列表。
- 动画遵守减少动态效果偏好。

### 隐私

- 首期不建立访客账号和个人画像。
- 定位功能需要浏览器明确授权；拒绝授权不影响手动搜索。
- 日志不保存不必要的精确访客位置。

### 数据边界

- 所有实时数据均可能存在延迟、遗漏或误差。
- 机场周边航班不等于到离港航班。
- 航线匹配不等于官方班次表。
- 产品不应用于飞行安全、空管或机场运行决策。
