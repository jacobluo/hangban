# 天气雷达图层设计

## 1. 文档定位

本文定义「航迹」首期天气雷达图层的产品范围、数据源、模块 seam、HTTP 契约、短期缓存、安全限制、界面状态和验证要求。

本设计已经完成范围与视觉方向确认。开始前端实现前，仍需先更新 Ardot `main_deep（16:1）` 中的筛选与图层面板、桌面地图和手机地图状态，并完成设计确认。

## 2. 目标

在现有 MapLibre 航班地图上增加可选的天气雷达叠加层，使访客能够同时查看航班实时位置和降水分布。

首期完成以下结果：

- 图层面板中的「天气雷达」由禁用项改为可切换项。
- 雷达图层默认关闭，由访客主动开启。
- 地图展示最新雷达帧、降水强度图例、帧时间和来源署名。
- 雷达供应商不可用时，航班地图和其他数据源继续工作。
- 浏览器只访问本项目 API，不直接依赖供应商地址或响应结构。
- 雷达元数据和用户访问过的瓦片最多缓存 24 小时，不建设长期档案。

## 3. 已确认范围

### 3.1 包含

- RainViewer 单数据源接入。
- 最新雷达帧叠加。
- 桌面端和手机端图层开关。
- 地图左下角紧凑图例。
- 最新、延迟、历史缓存和不可用状态。
- 24 小时有界短期缓存。
- 服务端瓦片代理、输入校验和稳定错误响应。
- RainViewer 可见署名。
- 默认离线自动化测试和显式公网冒烟检查。

### 3.2 暂不包含

- 雷达动画、天气时间轴和历史查询。
- 短时降水预报、风场、云层、温度和预警。
- 中国、东亚或全球区域的定时抓取。
- PostgreSQL、对象存储或数据湖天气档案。
- 多天气数据源融合和自动切换。
- 航班轨迹与天气历史的关联分析。
- 生产 SLA 或 1,000 并发天气瓦片容量承诺。

## 4. 数据源选择

### 4.1 首期选择

首期使用 RainViewer Weather Maps API。该接口提供 150 多个国家的雷达瓦片，当前公开元数据包含最近约 2 小时、10 分钟间隔的雷达帧，瓦片最大缩放级别为 7。

首期只选择最新帧，不使用过去帧和临近预报。界面必须显示 `Weather radar by RainViewer` 或语义等价的可见署名，并提供指向 RainViewer 的链接。

RainViewer 免费公共接口面向个人、教育和小型社区用途，不提供 SLA，也不适合高流量商业应用。该能力在产品中标记为实验性。投入公开商业生产前，必须重新核对授权、再分发、缓存和容量条件。

### 4.2 未选方案

- OpenWeather Weather Maps 1.0：免费申请 API Key，可提供全球降水瓦片，但不能稳定等同于实时雷达观测。本期不接入。
- Tomorrow.io：免费套餐按单张瓦片计费，每日和每小时额度不足以支撑交互式全球地图。本期不接入。
- 中国天气网 SmartWeatherAPI：官方能力包含雷达和云图，但需要邮件审核，公开页面未明确标准 XYZ 瓦片契约。可以并行申请，取得技术白皮书后另行评估。
- 和风天气：免费订阅包含分钟级降水预报，但不是全国雷达瓦片。本期不接入。
- 彩云天气：雷达图属于企业套餐增值服务，不符合首期免费数据源限制。

## 5. 模块设计

### 5.1 天气雷达契约

天气雷达是独立数据域，不进入航班位置候选、去重、融合或 `SourceStatus`。`packages/contracts` 定义供应商中立的雷达状态：

```ts
type WeatherRadarFreshness = 'latest' | 'delayed' | 'historical-cache';

type WeatherRadarStatus =
  | {
      available: true;
      providerId: 'rainviewer';
      frameId: string;
      frameTime: string;
      freshness: WeatherRadarFreshness;
      tileTemplate: string;
      attribution: {
        label: string;
        url: string;
      };
    }
  | {
      available: false;
      providerId: 'rainviewer';
      reason: 'DISABLED' | 'UPSTREAM_UNAVAILABLE' | 'NO_VALID_FRAME' | 'FRAME_EXPIRED';
    };
```

`tileTemplate` 只包含本项目同源 API 地址。契约不得暴露 RainViewer 的 `host` 或原始 `path`。

### 5.2 RainViewer 适配器

`packages/adapters` 增加天气雷达适配器，负责：

- 请求并校验 `weather-maps.json`。
- 从 `radar.past` 选择时间最新的有效帧。
- 将供应商帧路径映射为内部不透明 `frameId`。
- 只允许 RainViewer 配置的固定 HTTPS 主机。
- 请求瓦片时固定尺寸、颜色方案和平滑选项。
- 将超时、无效 JSON、无有效帧、非 PNG 和上游错误转换为稳定错误类型。

适配器不生成 Fastify 响应，不管理 MapLibre 图层，也不修改航班来源状态。

### 5.3 天气雷达模块

API 内部天气雷达模块封装以下行为：

- 获取当前统一状态。
- 维护供应商帧与内部 `frameId` 的受控映射。
- 读取和写入元数据缓存与瓦片缓存。
- 根据当前时间计算新鲜度。
- 在上游失败时选择仍处于 24 小时范围内的最近缓存帧。
- 执行缓存容量限制和淘汰。

模块向 Fastify 路由提供小型接口。路由不直接拼接供应商 URL，不理解 RainViewer 响应结构。

## 6. HTTP 接口

### 6.1 雷达状态

```text
GET /api/v1/weather/radar
```

行为：

- 功能关闭时返回 `available: false` 和 `DISABLED`。
- 有最新或缓存帧时返回统一状态和同源 `tileTemplate`。
- 无有效帧时返回 `available: false`，不返回供应商错误正文。
- 响应体使用 Zod 契约校验。

### 6.2 雷达瓦片

```text
GET /api/v1/weather/radar/tiles/:frameId/:z/:x/:y.png
```

行为：

- `frameId` 必须存在于服务端受控映射或有效缓存索引中。
- `z` 限制在 RainViewer 支持范围内，最大值为 7。
- `x` 和 `y` 必须满足对应缩放级别的 XYZ 范围。
- 只接受经过验证的 PNG 响应，并限制响应体大小和请求时间。
- 缓存命中时不访问上游。
- 响应设置适合 CDN 的缓存头，但不得超过帧缓存剩余 TTL。
- 未找到、无效参数和上游故障使用稳定错误码。

该接口不能接受 URL、主机名或任意路径参数，不能用作通用代理。

## 7. 缓存和新鲜度

### 7.1 缓存边界

- 雷达帧元数据和用户实际访问过的瓦片最多缓存 24 小时。
- 缓存受条目数和总字节数双重上限约束；达到容量上限时允许提前淘汰最久未使用条目。
- 不主动抓取未被地图请求的瓦片。
- 不写入 PostgreSQL 或长期对象存储。
- 不保证进程重启后恢复缓存。
- 多 API 实例可以依赖 CDN 分担重复瓦片请求；首期不要求在 Redis 中保存大量 PNG 二进制内容。

24 小时表示最大缓存 TTL，不表示历史数据完整保留。缓存的目的只是减少重复请求并支持短时故障降级。

### 7.2 新鲜度语义

以 RainViewer 帧时间和服务端当前时间计算：

| 数据年龄       | 状态               | 界面行为                                      |
| -------------- | ------------------ | --------------------------------------------- |
| 0–15 分钟      | `latest`           | 显示「最新」                                  |
| 15 分钟–2 小时 | `delayed`          | 显示「数据延迟」                              |
| 2–24 小时      | `historical-cache` | 显示「非当前天气 · X 小时前」，降低图层透明度 |
| 超过 24 小时   | 不可用             | 停止展示并自动关闭天气雷达图层                |

RainViewer 的帧时间代表拼图帧生成时间，不保证每个像素的观测时间完全一致。界面文案使用「雷达帧时间」，不表述为所有区域的精确观测时间。

## 8. 界面设计

### 8.1 图层开关

- 复用现有「筛选与图层」面板。
- 「天气雷达」默认关闭。
- 开启时先请求雷达状态，加载不阻塞地图和航班交互。
- 加载失败时恢复关闭状态，并显示「天气雷达暂时不可用」。
- 开关状态只保存在当前浏览器会话，不建设账号同步。

### 8.2 地图层级

天气雷达使用 MapLibre raster source 和 raster layer。层级从下到上为：

1. 地图底图。
2. 天气雷达。
3. 机场。
4. 航迹和航线。
5. 航班和选中态。

`latest` 和 `delayed` 默认透明度约为 55%。`historical-cache` 进一步降低透明度，避免旧雷达图像被误认为当前天气。航班标记、选中光环和航线始终保持可辨识。

### 8.3 紧凑图例

开启成功后，在地图左下角展示紧凑图例：

- 标题「天气雷达」。
- 从弱到强的降水色标。
- 雷达帧时间和新鲜度文案。
- `Weather radar by RainViewer` 可见署名和链接。

桌面端使用约 240 px 宽的浮层。手机端缩小宽度，并在短时位置回看控件出现时上移，不能遮挡回看、地图定位或主要触控区域。

### 8.4 Ardot 设计门

修改前端前，必须在 `main_deep（16:1）` 补齐：

- 桌面端图层面板开启状态。
- 桌面地图紧凑图例和雷达层级。
- 手机端图层面板开启状态。
- 手机地图紧凑图例以及与回看控件并存状态。
- 加载、数据延迟、历史缓存和不可用状态。

设计确认后才能修改 Next.js 与 MapLibre 实现。

## 9. 配置

建议新增以下服务端配置：

```text
WEATHER_RADAR_ENABLED
RAINVIEWER_BASE_URL
WEATHER_RADAR_TIMEOUT_MS
WEATHER_RADAR_CACHE_TTL_MS
WEATHER_RADAR_CACHE_MAX_ENTRIES
WEATHER_RADAR_CACHE_MAX_BYTES
WEATHER_RADAR_MAX_ZOOM
```

约束：

- `WEATHER_RADAR_CACHE_TTL_MS` 默认值为 24 小时。
- `WEATHER_RADAR_ENABLED` 默认值为 `true`，仅用于运维侧显式关闭服务端天气能力。
- `WEATHER_RADAR_MAX_ZOOM` 不得超过 7。
- `RAINVIEWER_BASE_URL` 只由服务端读取，并限制为 HTTPS URL。
- 首期不需要 API Key。
- 天气雷达与 `DATA_MODE=demo|live` 解耦；服务端默认启用，前端地图图层仍默认关闭。

## 10. 故障处理

- 上游元数据请求失败：优先使用不超过 24 小时的缓存帧，并按年龄标记状态。
- 瓦片请求失败：不影响其他瓦片、航班和机场图层；地图保持可操作。
- 缓存帧超过 24 小时：返回不可用状态并关闭前端图层。
- 无雷达覆盖：允许返回透明瓦片，不将其解释为服务故障。
- 非 PNG、响应过大或响应校验失败：拒绝缓存并记录稳定原因码。
- 配置关闭：不发起 RainViewer 网络请求。

日志不得记录完整上游响应正文。指标和日志不得把天气雷达故障合并为航班数据源故障。

## 11. 测试与验证

### 11.1 单元测试

- RainViewer 元数据解析和最新帧选择。
- 空帧、无效时间、无效路径和无效响应。
- 新鲜度在 15 分钟、2 小时和 24 小时边界的分类。
- LRU、TTL、条目数和字节数限制。
- 缓存帧降级和超过 24 小时淘汰。
- XYZ 坐标范围和最大缩放级别校验。

### 11.2 API 测试

- 功能关闭、最新、延迟、历史缓存和不可用响应。
- 同源瓦片模板不暴露供应商主机。
- PNG 代理、缓存命中和 CDN 缓存头。
- 非法 `frameId`、缩放级别和坐标。
- 超时、非 PNG、响应过大和上游错误。
- 雷达错误不改变航班来源状态或航班接口结果。

### 11.3 前端组件测试

- 雷达默认关闭。
- 加载期间开关状态。
- 开启后创建 raster source、raster layer 和紧凑图例。
- 图层位于底图之上、航班之下。
- 最新、延迟和历史缓存文案。
- 失败后关闭并显示安全错误提示。
- RainViewer 署名可访问。

### 11.4 E2E 与视觉检查

- 桌面端开启天气雷达并查看图例。
- 手机端开启天气雷达，图例不遮挡主要控件。
- 历史缓存状态明确显示「非当前天气」。
- 雷达接口失败时航班仍正常显示和更新。
- 使用 1440 × 900 和 390 × 844 视口完成视觉核对。

默认测试全部 mock 上游网络，不访问公网。可以新增显式 RainViewer 冒烟命令，只验证元数据契约和一张瓦片，不纳入默认 `pnpm verify`。

## 12. 发布条件

首期发布前必须完成：

- Ardot 状态更新和设计确认。
- RainViewer 最新使用条款、署名和免费用途复核。
- 服务端代理的 SSRF、响应大小和超时检查。
- 缓存容量上限验证。
- 桌面端和手机端视觉检查。
- 聚焦测试、`pnpm verify` 和 `pnpm format:check`。

免费 RainViewer 能力不计入当前 1,000 并发生产容量承诺。进入高流量或商业生产前，需要获得相应授权，或替换为具有明确生产许可和 SLA 的天气数据源。
