# 全球实时航班 MVP 实施规格

## 目标

实现可本地运行、可测试、可继续接入生产数据的「航迹」MVP。系统向未登录访客提供全球实时地图、航班详情、机场探索和航线探索，并通过数据适配层屏蔽公开数据源差异。

## 交付范围

- pnpm workspace，包含 `web`、`api`、`ingestor` 和共享 packages。
- 统一的航班、机场、航线和数据源状态契约。
- ADSB.lol、OpenSky、Airplanes.live 适配器边界；默认使用稳定的演示适配器保证本地可运行。
- 航班规范化、去重、融合、新鲜度和覆盖度计算。
- Fastify 搜索、航班、机场、航线、地图快照和数据源状态 API。
- 按地图边界订阅的 WebSocket 首次快照与增量事件。
- Next.js + MapLibre GL JS 响应式界面，覆盖 Ardot 已确认的桌面端与手机端核心流程。
- Vitest 单元/组件测试与 Playwright 核心流程测试。

## 实施边界

- 不实现账号、收藏、支付和个性化。
- 不实现官方到港板、离港板、登机口、延误预测或订票。
- 不在首期保存长期航迹。
- PostgreSQL、PostGIS 和 Redis 通过可替换端口定义；本地 MVP 使用内存实现，生产部署再启用外部基础设施。
- 浏览器不直接调用第三方航班数据源。

## 运行形态

采用方案 1：模块化单体 API 与独立采集进程。共享领域包定义稳定接口，本地开发可由 API 内嵌演示数据仓库启动；采集进程可独立运行并向状态仓库写入统一记录。

## 核心数据流

1. Provider adapter 获取并校验供应商快照。
2. Normalizer 转换时间、单位和缺失字段。
3. Fusion engine 按 ICAO 24 位地址去重，并按新鲜度和字段质量合并。
4. Latest-flight repository 保存当前状态并生成增量。
5. HTTP API 提供查询与首次快照。
6. WebSocket hub 按订阅边界过滤并推送增量。
7. Web 客户端使用 MapLibre 图层渲染航班，并用面板或底部抽屉展示对象详情。

## 质量要求

- 外部输入使用 Zod 校验。
- 领域纯函数和错误分支有单元测试。
- API 使用注入式仓库测试，不访问真实网络。
- 核心界面具有正常、空数据、过期和降级状态。
- 桌面端和 390 px 手机视口无关键内容溢出。
- 完整验证命令为 `pnpm verify`。

## 完成定义

- `pnpm install` 后可启动 Web 与 API。
- `pnpm verify` 通过 lint、类型检查、单元/组件测试、构建和 Playwright 测试。
- 首页地图、搜索、航班详情、机场探索和航线探索可使用演示数据完成完整交互。
- API 与 WebSocket 契约有自动化测试。
- README、AGENTS 和架构文档与实际目录及命令一致。
