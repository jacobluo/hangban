# 地图交互细节优化设计

## 1. 背景

当前地图界面存在四个相互独立但都影响定位与返回效率的问题：选中机场与普通机场标记过于接近；航线页面的机场选择器被左侧面板裁切；天气雷达图例标题没有直接说明展示的是降水；从机场周边航班进入航班详情后会切换到全球实时页面，关闭详情无法恢复原机场上下文。

本轮只修复这些界面与交互问题，不修改航迹回看。当前航迹回看仍是依据最新位置、航向与地速生成的 0–15 分钟估算位置，不扩展时长，也不增加自动播放。

## 2. 目标

- 让选中机场在地图上可以一眼识别，同时不改变普通机场的视觉权重。
- 让航线页面的机场选择器完整显示，并在内容较多时由列表内部滚动。
- 把地图雷达图例标题改为更准确的「天气雷达降水图」。
- 从机场周边航班进入航班详情后，提供明确且可预期的返回路径，并恢复原机场上下文。

## 3. 非目标

- 不修改机场、航班、航线或天气数据契约。
- 不修改图层开关、数据状态、错误提示中的「天气雷达」术语。
- 不引入通用路由栈、浏览器 URL 路由或跨刷新导航恢复。
- 不修改轨迹历史存储、估算算法、回看时长或播放行为。

## 4. 设计

### 4.1 选中机场焦点

普通机场继续使用现有白色圆点和品牌蓝描边。选中机场改为 9 px 橙色实心圆、2 px 白色描边，并增加约 18 px 的半透明橙色静态光环。机场代码使用橙色强调，并通过独立的选中标签图层在所有地图缩放级别保持可见。

选中态同时使用颜色、尺寸、描边、光环和标签区分，不依赖持续动画。光环位于机场点下方，标签位于机场点上方；航班点和航迹继续保持现有层级。

### 4.2 航线机场选择器防裁切

机场选择器打开时，航线构建面板增加显式打开状态。该状态允许选择器越过面板的滚动边界，避免被 `overflow-y: auto` 裁切。选择器本身继续使用有界高度，选项列表独立滚动。

桌面端选择器不得超出地图可视区域底部；手机端继续使用较短的视口相对最大高度。关闭选择器后，航线构建面板恢复原有滚动约束。提高 `z-index` 不是本问题的修复方式，因为当前问题来自父级裁切。

### 4.3 天气雷达图例标题

地图左下角图例的可见标题从「天气雷达」改为「天气雷达降水图」。图层开关仍称为「天气雷达」，图例区域的可访问名称仍为「天气雷达图例」，强度色标、帧时间、历史缓存说明和 RainViewer 署名均保持不变。

### 4.4 从机场周边航班返回

从机场详情的「周边实时航班」选择航班时，应用记录来源机场，然后沿用现有行为切换到全球实时地图并展示航班详情。航班详情头部增加上下文动作「返回 {机场代码} 周边航班」。

点击上下文返回动作或右上角关闭按钮时，行为一致：

1. 关闭当前航班详情。
2. 恢复机场页面。
3. 恢复原先选中的机场和手机端机场详情状态。
4. 地图重新聚焦该机场。

从全球搜索、地图航班点或航线页面进入航班详情时，不建立机场返回上下文，关闭详情仍保持现有全球实时行为。用户手动切换顶部页面、选择其他机场或从其他入口选择航班时，清除旧的机场返回上下文，避免返回到过期页面。

若从该航班摘要继续打开完整详情，「返回地图」先返回航班摘要；随后仍可使用机场上下文返回动作恢复原机场。

## 5. 响应式与可访问性

- 桌面端 1440 × 900 和手机端 390 × 844 均需验证机场焦点、选择器边界和返回动作。
- 上下文返回动作使用真实按钮，并具有包含机场代码的可访问名称。
- 关闭按钮的可访问名称继续为「关闭航班详情」，但在存在机场上下文时执行恢复机场的行为。
- 机场选择器仍支持 Escape 关闭、搜索输入自动聚焦和列表内部滚动。
- 选中机场不使用脉冲动画，不增加 `prefers-reduced-motion` 分支。

## 6. 测试与验收

- MapLibre 图层测试验证选中机场光环、选中点和独立标签图层的过滤条件与关键样式。
- 组件测试验证航线机场选择器打开状态，以及列表不再依赖父面板滚动区域展示。
- 天气雷达图例测试验证可见标题为「天气雷达降水图」，其他语义保持不变。
- `AppShell` 组件测试验证从机场周边航班进入详情后出现上下文返回动作，点击返回动作和关闭按钮都恢复同一机场。
- Playwright 覆盖桌面端机场往返流程、航线选择器完整展示和手机端机场往返流程。
- 聚焦测试通过后运行 `pnpm verify` 与 `pnpm format:check`，并完成两个目标视口的视觉核对。

## 7. 原型与实施边界

选中机场应具有明确焦点，已属于 `main_deep（16:1）` 的确认原则；本轮需要在对应画板补齐具体标记状态。航线选择器裁切属于实现缺陷，应按现有选择器原型还原。天气雷达标题和机场上下文返回动作需要在对应桌面端与手机端画板同步后再修改前端实现，并在视觉检查记录中说明。
