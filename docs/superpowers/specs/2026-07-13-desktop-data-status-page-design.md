# 桌面端数据状态整页修复设计

## 1. 背景

Ardot `main_deep（16:1）` 中的 `Deep / Desktop / Data Coverage and Status（16:1970）` 是 `1440 × 900` 的完整页面，包含全局顶栏、状态摘要、数据源列表和覆盖说明。

当前实现点击「实时数据」后，在地图右侧打开约 `360 px` 宽的 `DataStatusPanel`。该实现复用了地图对象详情侧栏，与已确认的桌面端设计不一致，也压缩了数据源状态和覆盖说明的阅读空间。

## 2. 目标

- PC 端点击实时状态入口后，展示独立的数据覆盖与服务状态整页。
- 手机端继续使用全高、可滚动的状态页面形态，不恢复为半高抽屉。
- 返回地图后保留打开状态页前的地图视野、搜索、筛选和选中对象。
- 页面只展示统一领域模型提供的真实状态，不复刻 Ardot 中的示意航班数、延迟或覆盖率。

## 3. 不包含

- 不新增账号、设置或管理能力。
- 不修改数据适配层、实时事件、数据库或数据源契约。
- 不新增可分享的 URL 路由或浏览器历史记录。
- 不新增后端计算的全球航线覆盖率；缺少可靠数据时不展示该指标。
- 不调整筛选与图层面板、航班详情侧栏或机场与航线页面。

## 4. 方案比较

### 4.1 采用方案：应用内整页状态

在 `AppShell` 中使用独立的页面开关，与完整航班详情处于同一层级。状态页面离开地图舞台渲染，复用当前实时状态数据和重试回调。

优点：符合现有单页状态架构；返回时可保留地图上下文；无需引入路由和数据重新加载。

### 4.2 未采用：新增 Next.js 路由

新增 `/status` 路由能够提供可分享地址，但需要提升实时状态数据的作用域，并处理路由返回时的地图状态恢复。当前产品没有状态页直达或分享要求，因此暂不引入。

### 4.3 未采用：扩大现有右侧面板

将侧栏扩展为大面板改动较少，但仍属于地图叠层，无法还原 Ardot 的完整页面层级，也不能解决桌面端内容密度问题。

## 5. 页面结构

### 5.1 桌面端

- 保留 `64 px` 全局顶栏和「全球实时 / 机场 / 航线」导航。
- 状态入口保持当前激活语义，不展示 Ardot 中无法验证的全球航班总数。
- 主体使用浅色页面背景，内容宽度与 Ardot 的 `1440 × 900` 画板比例一致。
- 页面标题为「数据覆盖与服务状态」，副标题说明数据源、覆盖范围和数据新鲜度。
- 顶部依次展示总体健康状态、正常来源数、当前已获得航班数和最后成功时间。
- 主体左侧展示数据源列表，右侧展示覆盖边界、实时定义和可执行操作。
- 提供「返回地图」操作；按 `Escape` 也返回地图。

当前契约无法提供平均延迟和各区域覆盖率，因此不展示 Ardot 中的 `8 s`、`87%` 或各区域百分比。当前航班数必须附带「仅反映当前数据覆盖」说明。

### 5.2 手机端

- 页面占满可用视口并允许纵向滚动。
- 顶部提供返回地图操作和页面标题。
- 状态摘要改为单列或双列紧凑布局，数据源卡片纵向排列。
- 主要操作触控区域不小于 `44 × 44 px`。
- 页面不得扩大文档宽度，不得与地图控制同时叠加显示。

## 6. 状态与数据

页面继续消费现有输入：

- `SourceStatus[]`
- `RealtimeConnectionState`
- 当前已获得航班数
- 最近更新时间
- 重试回调

总体状态规则保持不变：

- 连接正常且所有来源健康：显示「全部服务正常」。
- 连接正常但存在降级来源：显示「部分服务降级」。
- 首次连接：显示「正在连接实时服务」。
- 重连中：显示「正在重新连接」。
- 连接中断：显示「实时连接已中断」。

每个数据源继续展示最近结果、最后成功时间、记录数、稳定错误类型和缓存语义，不输出内部错误栈或供应商凭据。

## 7. 交互与可访问性

- 打开状态页后，地图主区域不再处于可交互层级。
- 页面标题通过 `aria-labelledby` 与页面容器关联。
- 「返回地图」按钮打开页面后获得焦点。
- `Escape` 返回地图。
- 状态不能只通过颜色表达，继续保留文字和结构差异。
- 返回地图后，将焦点恢复到触发状态页的实时状态入口。

## 8. 测试与验收

### 8.1 组件测试

- 点击实时状态入口后，状态内容出现在独立页面容器中。
- 地图主区域在状态页打开时不可交互或不在可访问树中。
- 点击「返回地图」或按 `Escape` 后恢复地图。
- 数据源、覆盖提示、错误类型和重试操作继续可用。

### 8.2 E2E

- PC `1440 × 900`：状态页占据应用内容区，不存在右侧 `360 px` 侧栏形态。
- 手机 `390 × 844`：状态页占满可用视口、可滚动且无水平溢出。
- 返回地图后，打开状态页前的选中对象和筛选状态保持不变。

### 8.3 完成门禁

- 聚焦组件测试通过。
- 相关 Playwright 流程通过。
- `pnpm verify` 通过。
- `pnpm format:check` 通过。
